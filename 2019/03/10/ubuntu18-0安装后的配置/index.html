ubuntu18.0安装后的配置

Posted on 2019-03-10 | Comments: | Views:

新立得软件包管理系统

使用 Ubuntu 软件中心无法让您安装和删除某些更高级的软件包，比如 Apache 网络服务、PHP 编程语言或 Scribus。这时，就得用新立得软件包管理器。

可以使用新立得来安装、删除、配置或升级软件包，浏览、排序和搜索可用软件包列表，管理仓库或升级整个系统。您可以在执行更改前将一系列动作加入操作队列。新立得会通知您安装您所选软件包需要的额外软件包，以及与您计算机上已安装软件包之间的冲突。除此之外，它还展示更多诸如软件包状态、来源和过滤器信息。

安装：sudo apt-get install synaptic

更新内核与删除内核

删除内核

  • 首先查看当前内核版本号，防止误删：uname -sr
  • 查看当前安装的内核：dpkg --get-selections|grep linux
  • 删除命令：例：apt-get purge linux-modules-4.15.0-29-generic
  • 有一些人使用apt-get remove删除的，导致删除不干净就用到：sudo dpkg -P linux-image-*.*.*-**-generic linux-image-extra-*.*.*-**-generic
  • PS：还有通过正则表达式直接完成的（大胆的试）：sudo dpkg -l 'linux-*' | sed '/^ii/!d;/'"$(uname -r | sed "s/\(.*\)-\([^0-9]\+\)/\1/")"'/d;s/^[^ ]* [^ ]* \([^ ]*\).*/\1/;/[0-9]/!d' | xargs sudo apt-get -y purge

  • 删除系统残留：apt-get autoremove &amp; apt-get autoclean
    更新内核

  • 下载内核：http://kernel.ubuntu.com/~kernel-ppa/mainline/找到相应的内核版本，找到相应的文件夹，下载内核包（根据系统结构选择相应版本，amd64对应64位系统，i386对应32位系统），

  • (最新版内核)linux-headers-4.18.11-041811_4.18.11-041811.201809290731_all.deb
    linux-headers-4.18.11-041811-generic_4.18.11-041811.201809290731_amd64.deb
    linux-image-unsigned-4.18.11-041811-generic_4.18.11-041811.201809290731_amd64.deb
    linux-modules-4.18.11-041811-generic_4.18.11-041811.201809290731_amd64.deb(注意这里没有全都下载)
  • 转到文件夹，安装：sudo dpkg -i *.deb
  • 重启，uname -sr查看是否是最新
  • https://blog.csdn.net/CSDN_duomaomao/article/details/77668946（由此得）

  • 记得更新：update-grup(否则可能会重新更新回来旧内核)

    通讯工具的添加1.0

  • 这里选用的是wineqq，首先安装wine环境https://wiki.winehq.org/Ubuntu贴上官方文档，这里我在复述一遍

  • 如果系统是64位的：sudo dpkg --add-architecture i386
  • 添加源：
  • On Linux Mint 18.x, the last line should be the following : sudo apt-add-repository 'deb https://dl.winehq.org/wine-builds/ubuntu/ xenial main'（这个linux mint 18是基于linux 4.4的）
  • 更新：apt-get update
  • 安装：sudo apt-get install --install-recommends winehq-devel（有三个版本从上往下逐渐偏于不稳定）
  • 安装wineqq：下载地址：https://pan.baidu.com/s/1KAP1h4qVDBi_LY4lBrfCYQ 提取密码：4854
  • 解压到当前用户主目录：tar xvf wineQQ8.9_19990.tar.xz -C ~/

  • 不出意外就安装完成了，贴上另一种方法https://blog.csdn.net/lyc_yongcai/article/details/73250057，原来的方法可能会出现
    E: Malformed entry 52 in list file /etc/apt/sources.list (Component)
    E: The list of sources could not be read.

  • 解决办法：gedit /etc/apt/sources.list打开文件，把后面的第三方源全部注释掉就没事了，安装完之后没法输密码（经常崩溃= =）

  • 删除wine的软件或者进行操作时的目录：wine会在/home的用户目录生成三个隐藏的文件夹 .wine、.local、.config 等文件夹，快捷键 ctrl+H 可以显示出来;
  • exe文件的卸载：进入 .wine 文件夹可以看到 drive_c 文件夹，这是wine自动生成的虚拟windows C盘，里面有类似windows系统盘的目录结构，在里面找到需要卸载的软件文件夹删除即可；
    找到/home/用户名/.local/share/applications/wine/Programs，将软件对应的文件删除；
    找到/home/用户名/.config/menus/applications-merged，将软件对应的文件删除；
    这时候已经删除完毕，但是可能还会看到桌面图标或软件列表，重启系统即可。

  • 执行apt-get update后出现仓库***没有Release文件：在etc/apt/sources.list.d目录中删除相应文件。（chomd+777+文件 代表赋予所用用户所有权限）
    卸载wine

  • 卸载wine主程序，在终端里输入：sudo apt-get remove --purge wine

  • 然后删除wine的目录文件：rm -r ~/.wine

  • 卸载残留不用的软件包：sudo apt-get autoremove
